Sobre

O portal SisGEO é uma plataforma web desenvolvida pela Universidade Federal de Alfenas(UNIFAL-MG) em parceria com o INPE para acesso, consulta, análise e disseminação de dados geográficos gerados elos projetos de monitoramento do Lago de Furnas. Através do SisGeo qualquer pessoa pode acessar os mapas e realizar análises sobre os dados diretamente do portal, sem necessitar de ferramentas específicas.

A algumas décadas atrás, pelos idos da década de 1990, o compartilhamento de dados - sejam espaciais ou não - esbarrava na ineficiente ou até mesmo inexistente, estrutura de tecnologia de informação. Com o desenvolvimento e popularização de novos componentes de hardware e software em computadores pessoais, aliado ao investimento na infraestrutura de comunicações em servidores, plataformas de comunicações e redes, o tráfego de dados obteve um expressivo incremento. A maior referência com relação ao tráfego de dados em rede e conectividade entre empresas e pessoas foi, sem dúvida a internet, que popularizou-se no Brasil no início da década de 2000.

No Brasil, investimentos foram realizados a fim de se estruturar redes de alta velocidade e que conectassem institutos de pesquisas e órgãos das esferas administrativas. A Rede Nacional de Pesquisa (RNP) foi responsável por introduzir a internet em diversos locais interioranos no Brasil, realizando a inclusão digital de milhares de discentes nas instituições de ensino e pesquisa, possibilitando, o compartilhamento de dados e conhecimento. Entretanto, muitos dados são de difícil acesso à sociedade até os dias atuais e, tendo o foco de diminuir a distância entre os dados produzidos pelas instituições de ensino e pesquisa e a sociedade, o curso de Geografia por meio do Laboratório de Cartografia e o curso de Bacharelado em Ciência da Computação da Universidade Federal de Alfenas (UNIFAL-MG), desenvolvem em cooperação com o Instituto Nacional de Pesquisas Espaciais (INPE) e apoio do Núcleo de Tecnologia da Informação da UNIFAL-MG, o Sistema de Compartilhamento de Dados Geoespaciais (SisGEO).

O SisGEO possui como objetivo compartilhar dados espaciais produzidos pelo curso de Graduação e Pós-graduação em Geografia, nos mais diferentes recortes espaciais, por meio de estágios, iniciações científicas, trabalhos de conclusão de curso, dissertações de Mestrado e projetos de pesquisa realizadas(os). A coordenação do SisGEO está a cargo do Laboratório de Cartografia e do Grupo de Pesquisa em Geodinâmica de Bacias Hidrográficas (GeoHidro), que se responsabilizará pela atualização da plataforma e avaliação de dados a serem inseridos.

O SisGEO encontra-se em sua versão 1.0 que ofertará o armazenamento, visualização e disponibilização de dados espaciais, seguindo padrões internacionais de dados espaciais definidos pela Open Geospatial Consortium (OGC). Ressalta-se que toda a estrutura do sistema está embasada em tecnologia de software livre, o que reduz custos com licenças e/ou permissões para seu desenvolvimento.

Tecnologias

O SisGEO utiliza tecnologias gratuitas sob diferentes licenças de software:
- Leaflet: bilioteca Javacript para exibir mapas na web e dispositivos móveis. Licença: "BSD-2-Clause”.
- Angular 5: framework voltado ao desenvolvimento de aplicações web. Licença: "MIT license”.
- Docker: plataforma de criação e administração e ambientes e serviços, facilitando o gerenciamento dos serviços utilizadas pelo SisGEO. Licença: "Apache 2.0 license”.
- MongoDB: banco de dados orientado a documentos. Licença: verificar versão para saber qual é a licença.
- Nginx: web server, que também pode ser utilizado para proxy reverso e balanceamento de carga. Licença: "BSD-2-Clause”.
- Geoserver: servidor que permite compartilhar, editar e processar dados geoespaciais. Licença: "GNU General Public License”.

O SisGEO utiliza a plataforma "docker", responsável pela criação e administração de aplicações em ambientes isolados, sendo que cada ambiente reúne todos os recursos necessários para a execução de uma aplicação. Com ele fica muito mais simples implantar e escalonar a aplicação, entre outras facilidades. Nele, existe o conceito de containers, que é uma virtualização em nível do sistema operacional, como se fosse uma máquina virtual, porém apenas com os recursos necessários para aquela aplicação, sendo assim, muito mais leve.

Portainer é uma interface gráfica responsável por gerenciar os containers, realizando o deploy e escalonamento, além de facilitar a visualização dos logs. Além dele, temos outros três containers que participam diretamente do funcionamento da sistema: "Geoserver", "Business API" e "Webmap". O fluxo é basicamente o seguinte: quando o container Webmap recebe uma requisição, ele retorna uma página web que contém código gerado pelo framework Angular, código do biblioteca Leaflet para requisitar e desenhar as layers no navegador, além de outros.

Nesse momento, ele também faz requisições Http para a API REST: Business API que por sua vez requisita dados de layers que estão armazenados no banco de dados "mongo". Esses dados especificam quais layers serão exibidas no "sidebar" da ferramenta, ordem das layers, em qual vision a layer se encontra, quais são os links de download das layers, links de "getFeatureInfo", que retornam informações da layer, entre outros.

Quando o usuário clica no botão para exibir a layer, é feita uma requisição de "getMap" para o geoserver, que por sua vez retorna o arquivo correspondente a layer requisitada. Quando esse arquivo é baixado pelo navegador, a biblioteca em javascript, Leaflet, é responsável por desenhar a layer no mesmo.
